# 钉钉已经彻底 CLI 化了。

> 没有 CLI 的产品，在 Agent 时代迟早会沦为一个只能截图、点按钮、绕远路的旧世界遗迹。

**这篇文章最值得细想的，不是钉钉出了个 CLI，而是它在公开承认一件事：未来软件会有两套界面，一套给人看，一套给 AI 用。真正决定产品能不能进下一代工作流的，往往不是 GUI 漂不漂亮，而是 CLI 和 API 到底给没给透。**

## GUI 是翻译层，AI 不需要翻译

人类需要按钮、菜单、表单、动画，是因为人不直接理解系统能力，所以产品得把底层操作翻译成可点击的界面。但 AI 不需要这层翻译。它不关心按钮圆不圆、动效丝不丝滑，它只关心命令、参数、权限和结果。你还逼它走 GUI，本质上就是让本来会说人话的人，先去看图猜谜。

这就是为什么 CLI 会在 AI 时代重新变成硬通货。它不是 DOS 时代那种给人背命令的老古董，而是给 Agent 直接调用能力的母语接口。命令长一点无所谓，输出是 JSON 更好，关键是别绕。能一行命令解决的事，谁还愿意让 Agent 去截图、识别、点按钮、等页面刷新？那不是智能，是折磨。

## 谁更容易被 AI 调用，谁就更容易活下去

钉钉这次真正聪明的地方，不只是把操作 CLI 化，还顺手把安全边界、结构化输出、过程留痕和面向 AI 的文档体系一起补上。说白了，它在做的不是“给开发者一个新入口”，而是把自己从 GUI 工具升级成 AI 能直接接管的基础设施。

这背后是个很残酷的判断：以后很多产品竞争的，不只是人类用户体验，还有“Agent 可调用性”。同样一个服务，一个只能点点点，一个可以直接命令式调用，AI 会选谁，没有任何悬念。到那时候，没 CLI 的产品不是不好用，而是根本排不上桌。
