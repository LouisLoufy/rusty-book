# 如果让我在 2026 年重新学机器学习

## 如果可以从头来过

![](https://cdn.jsdelivr.net/gh/beatai-org/beatai-assets@d636560ddb58a0d75173d1977cf7a323f1319997/ai-insights/2026-05/27/images/how-id-learn-machine-learning-in-2026/01.jpg)
*图片来源：Aron Visuals on Unsplash*

我在 Twitch 做 Senior Applied Scientist，过去七年一直在把 ML 系统部署到生产环境。**如果让我今天，在 2026 年从零开始重新学一遍，我几乎不会再按当年那条路走**。

说实话，现在网上大多数 ML 求职建议我都会无视——很多内容是在传授让**作者本人**在 2019 年拿到 offer 的那条路。那条路今天不管用了。这一点我有发言权：辅导过 200 多个人进入这个领域，亲眼看着事情变化了多少。

下面把真正会走的路一步一步讲清楚。如果过去三个月你一直在啃微积分课本，看完这篇可能会想重新规划一下。

## 直觉 > 数学

当然，我自己第一次学的时候就是这么干的——还没碰过任何模型，先在数学上花了几个月。把数学书从头读到尾，手推链式法则，全套来一遍。最后……是学会了。

但比真正需要的时间多花了好几年。更奇怪的是：**那一整套严谨训练，并没有真的让我对模型在实际中为什么这样表现建立起过硬的直觉**。梯度下降背后的数学我能讲清楚，但模型在生产环境里出问题时，对原因还是基本靠猜。

我不是说数学不重要。当然重要。但如果有人告诉你必须先把微积分学透才能做有用的 ML 工作，这个人要么是在读 PhD，要么是在转述**别人对他说过的话**。

**实际需要的数学比大家说得少得多**。你需要的是：

-   一点线性代数，足够让你理解矩阵相乘时到底在做什么。
-   够用的概率和统计，让 Bayes 定理、概率分布这些不再吓人。
-   把微积分学到链式法则，让反向传播说得通。

差不多就这些。

其他的等遇到具体问题再补就行。

那如果不啃课本，到底从哪入手？

我会从第一次学时绕过、后来又回过头补的那件事开始：**对这些算法到底在做什么、为什么这么做，建立真正的直觉**。

起点是 3Blue1Brown 的 [Essence of Linear Algebra](https://www.youtube.com/watch?v=fNk_zzaMoSs&list=PLZHQObOWTQDPD3MizzM2xVFitgF8hE_ab) 和 [Essence of Calculus](https://www.youtube.com/watch?v=WUvTyaaNkzM) 两个系列。他们把数学动画化，能真正看到一次矩阵变换对一个向量做了什么、导数在几何上长什么样。

概率和统计可以看 [StatQuest](https://www.youtube.com/watch?v=pYxNSUDSFH4)。Josh Starmer 擅长用一些可爱的小图把复杂话题讲明白，真的能让人听懂。

关键在于：**这一步不是要精通什么**，只是搭一张粗略的脑内地图，为下一步做准备。

这一步我会限制在两到四周内，避免陷入"看几个月视频、感觉学得挺顺，但其实一行代码都没动手写"的陷阱。

## 传统 ML 基础

等数学不再让你发怵，下一步要补的是**对传统机器学习的整体认知**。比如：分类和回归有什么区别？树模型、线性模型、神经网络各自在做什么？还是那句话，这一步不是要钻深——只要别人说"随机森林"时脑子里能浮现一个大致的图像就够了。这一阶段 [Andrew Ng's ML Specialization](https://imp.i384100.net/o44Kq9) 很合适；想找免费资源，StatQuest 也有覆盖主要算法的精彩视频。

之后花点时间学**神经网络基础**。还是先看老朋友 3Blue1Brown，再看 StatQuest，或者 [Andrew Ng's Deep Learning specialization](https://imp.i384100.net/bOOBax)，都是稳妥的选择。

有了这层底子之后，机器学习水平有一步性价比极高的提升——但前提是先会 Python。

## ML 用的 Python

机器学习的默认语言是 Python。scikit-learn、PyTorch、TensorFlow，以及基本上会用到的所有主流库都建在它之上。即便撇开 ML，早学 Python 也是聪明之举：**万一最后爱上写代码、却不爱数学，软件工程、数据工程，以及一堆相邻方向的大门照样会为你敞开**。

这一步不需要花哨内容。学好数据类型、控制流、函数、文件读写就够了。另外要把 NumPy 的矩阵运算和 pandas 的表格数据处理用熟——开始动手建模型后，这两件事就是日常。

大多数人到这一步就打开 scikit-learn、对一个模型调用 .fit()、拿到结果，然后觉得自己懂了。

但如果你跟当年的我一样，**其实根本没真正搞清楚发生了什么**。我后来想通过读教科书补课，效果有那么一点，但更像是在背书里的字句，而不是真的理解。

## 我最喜欢的学习诀窍

最终让我学透的方法，是用 numpy 从零开始实现算法。**先把当时能理解到的步骤、基础数学写下来，再一行一行写成代码**。逻辑回归、k-means、决策树这些东西，经过这个过程后清晰了不止一个量级。

当你能看到数据在算法的每一步里如何流动，数学就不再只是为了做题的数学。

这个过程同时是直接的面试准备：ML coding 面试经常就要求用 numpy 现场实现这些算法，提前练等于抢跑。

但现实是，前面讲的所有内容，**单靠它们没法让你拿到面试**。

**辅导过 200 多人完成这条转型路，看到的是同一种剧本反复上演**。有人按部就班——课全上了、材料都看了——但简历这一关就过不去。然后来私信问我：到底哪里做错了？

而另一个人，可能理论功底还更薄一点，反倒在同一批公司开始收到回复。对学得更多的那个人来说，这画面挺让人困惑。

差距几乎总能归结到两件事。第一件——也是马上要讲的这一件——是 GitHub 上有什么。第二件后面再讲，说实话可能更重要。先说 GitHub。

## 做真正的项目

大多数情况下，"学得更多"的那个人，仓库里塞满了课程练习和 Kaggle notebook——问题、数据，连评估指标都是别人喂到嘴边的。招聘经理看完，几乎得不到任何关于这人能不能胜任工作的信息。

**招聘经理需要看到的，不是你学过什么，而是真的能干这份工作上要干的事**。

有人手把手告诉你"做这个"的 capstone 项目，初学时挺好；但要走到职业水准，就得做职业水准的东西。**也就是：自己识别一个问题，找到并整理独有的数据，系统地评估候选模型，最后真的把模型部署到生产环境**。

举个例子。假设有个朋友开了家小型网店，用户老流失——注册之后不到一个月就消失。可以用他们的交易和互动数据搭一个流失预测模型，弄清楚到底哪些特征真的能预测流失，再给他们一个简单的仪表盘，标出有流失风险的用户。真实世界的数据、真实的用户（不灵的时候会直接告诉你），还有一堆要自己拍板的决定。

或者喜欢健身——可以做一个膳食规划 app，输入饮食限制，输出一周食谱外加买菜预算估算。范围小没关系，但每一个决定都是自己做的，最后能拿出真的可以演示的东西。

**真正能让求职打分上去的项目，最关键的是展示工程化能力**。一个跑出好结果的 Jupyter notebook 是起点。但一个用 Docker 容器化、部署在 AWS 或 GCP 上、配了基础 CI/CD 流水线、用 MLflow 或 Weights & Biases 跟踪实验、并持续监控模型表现的项目，才能让招聘经理相信你真的能在专业环境下干活。

越来越多在生产环境里构建的东西，已经不是经典 ML 应用，而更聚焦在 Gen AI 上。

## Gen AI

如果这套建议是半年前讲的，我大概会花更多篇幅给经典 ML 撑腰，告诉你先打牢经典基础再去操心 Gen AI；但最近想法变了一些。

经典 ML 仍然重要。模型评估这类底层逻辑要靠这层基础；如果做的是金融或医疗这类行业，还得理解更具可解释性的模型。

不是叫你跳过这些，前面讲的内容也都还成立。

但坦白讲，连反欺诈、推荐系统这种领域，也越来越往 Gen AI 路线上靠。**2026 年的 ML 工程师，既要懂经典 ML，也要懂 AI engineering**。

具体落到实操，**最大的两块是 RAG 和 evals**。RAG 不只是搭个检索流水线那么简单，关键在于知道什么时候该选 RAG、什么时候该选 fine-tuning、什么时候只是 prompt 没写好，以及怎么衡量这些方案是不是真的有效。

这就引出 evals——为 Gen AI 系统搭评估流水线，现在是工作里很大一块，而大多数人完全不知道怎么把它做好。这两块之外，还要懂 agent 和工具调用、面向生产系统的 prompt engineering、在给定成本和延迟预算下选对模型，以及 prompt injection 之类的安全基础。这些我在别的视频里讲得更深，但就是我会优先投入的几个领域。

做项目的过程中，要用 AI 辅助。**ML 工程师现在日常都在用 AI 编程助手，你得真的用得好**。这不只是在 Cursor 里开个对话窗口聊天那么简单——还要熟悉 agentic coding 的最佳实践，比如怎么并行高效地协作多个 agent。

但与此同时，**绝对不能把思考外包给 ChatGPT**。这里有一种"流畅性错觉"：AI 把一个完美答案递到面前，看完觉得自己懂了，但实际并没有完成那段把知识刻进脑子的认知工作。用 AI 解释卡住的概念、用 AI 自测，这些都行；但别让它写代码而你只是边点头边假装看懂。

## 面试准备

既然讲到代码——**我也希望能告诉你数据结构与算法面试对 ML 岗已经不重要了**。在某些公司这话确实成立，因为他们把 AI-first 的面试方式放在了优先级前面。但更多公司，尤其是大厂，仍然会安排一轮 LeetCode 式的题目，学习过程中要心里有数。

回到面试。前面说过收到回复和没收到回复的人之间有两个差别。第一件已经讲了，是作品集。第二件是——你认识谁。

好在**不必出身硅谷世家也能拿到内推**。可以主动建立线下和线上的人脉——参加会议、在网上社区里活跃、带着对欣赏对象工作的认真提问去联系他们。Networking 是提高成功率上单一杠杆最大的一件事。

这条路和我当年学的时候已经不一样了——更快、更务实，但在一个不断变化的市场里要覆盖的范围也更大。

并不轻松。**但完全做得到。一旦进入这个领域，我保证一切都值得。**
