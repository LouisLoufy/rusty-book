# AI 平台战争已经打响：四巨头争夺企业 AI 全栈

![](https://cdn.jsdelivr.net/gh/beatai-org/beatai-assets@d862590ea9216ae63f9cd9b4163d944ec90098a3/ai-insights/2026-06/04/images/the-ai-platform-wars-have-started/01.webp)

老戏码又上演了：一场争夺平台霸权的厮杀。这一回，赌注是 AI。

Anthropic 本周推出 Claude Managed Agents，是迄今最明确的信号——AI 市场已经进入新阶段。这不是又一次模型发布，而是 Anthropic 从「卖智能」转向「卖执行基础设施」：安全沙箱、长时会话、编排、治理，全部和 Claude Code 这类现有产品串到一起。

这是平台玩家的打法，不是模型供应商的打法。

IT 行业的格局向来由几个激烈厮杀的大平台塑造：Wintel 对 Mac，iOS 对 Android，PlayStation 对 Xbox 对 Nintendo Switch，AWS 对 GCP 对 Azure，名单还能一直列下去。

硅谷对平台的痴迷有个简单的理由：全世界最值钱的公司，清一色是平台公司。赢下一场平台竞赛回报极其丰厚，代价也极高。Mark Zuckerberg 砸下几百亿赌 VR 平台，至今没什么像样的成果——这背后的逻辑就在这儿。

而所有平台竞赛里，AI 这一场分量最重，尤其是对企业 AI 技术栈控制权的争夺。

## 从模型战到平台战

过去两年，AI 市场基本是一场模型战：推理更强、编码更强、多模态更强、跑分更高。但这从来都只是局部画面。

平台战的开端，是头部玩家不再只发零散组件，转而把上层的整套技术栈拼起来。眼下正在发生的就是这件事。Anthropic 进军托管运行时是一个明证；OpenAI 的 Frontier 愿景是另一个——它直接把未来描述成共享上下文、新人引导、反馈闭环、权限管理，以及一批跨真实业务系统运转的 AI 同事。

这套技术栈如今已经清晰可见，从底到顶大致是：

- **基础模型**：LLM、代码、视频、图像生成
- **agent 运行时**：托管执行、沙箱、工具调用、会话、浏览器或计算机访问、长时任务
- **上下文层**：企业搜索、连接器、记忆、带权限感知的检索、公司知识
- **构建 / 编排层**：工作流构建器、可复用 skill、agent 协同、评估、部署
- **应用层**：编码 agent、生产力 agent，以及作为入口的通用聊天界面
- **治理层**（横贯所有层，多数人仍低估它）：身份、权限、追踪、审计、安全、控制

眼下认真争夺整条栈的，是四家：OpenAI、Anthropic、Google、Microsoft。当然还有许多玩家在某一层提供零件，个别甚至在特定领域领先。

但历史给的经验是：真正要紧的，是整条平台栈的霸权之争。有资格下场的，只有这四巨头。

## 四巨头各自的牌面

![](https://cdn.jsdelivr.net/gh/beatai-org/beatai-assets@d862590ea9216ae63f9cd9b4163d944ec90098a3/ai-insights/2026-06/04/images/the-ai-platform-wars-have-started/02.webp)

**Anthropic** 大概是过去几个月转身幅度最大的一家。不久前，它还只是一家手握强悍前沿模型、一款高端聊天机器人、外加一个越做越强的编码产品的公司。直到推出 MCP 集成协议，它才第一次清晰露出平台野心。

接着是大幅扩展、定位为通用 agent 平台的 Claude Code，定位生产力工具的 Claude Cowork，以及现在的 agent 执行环境。Anthropic 的产品线仍有明显窟窿——比如缺图像和视频生成模型——但它在「企业真正看重什么」这件事上聚焦得相当好。

**OpenAI** 则可能仍把终局描绘得最清楚。Frontier 卖的不是一堆工具的集合，而是一个企业级平台：用共享上下文、明确权限和边界来构建、部署、管理 AI agent。

虽说在编码和 agent 领域最近被 Anthropic 反超，OpenAI 补栈的速度很快。Codex 已经是个像样的多 agent 阵地，能并行跑长时任务，且不止于编码；ChatGPT agent 把多种运行模式和连接器揉到一起。OpenAI 的运行时落地或许不如 Anthropic 清晰，但战略走向一目了然——它也想成为 AI 同事的操作系统，而且模型能力的覆盖面比 Anthropic 更宽。

四家里，**Google** 的技术面可能最广。云这边，Vertex 已经提供真正的 agent 运行时环境和大量其他企业组件；其上，Google 在搭一个像样的上下文与分发层，自然吃到了 Google Workspace 的红利；它有 agent 和工作流构建器，也有一批能打、但还不算热门的编码工具。Gemini 当然是款强劲的聊天产品，受益于模型质量，连带还有极受欢迎的图像、视频生成工具。

Google 的老毛病是产品战略容易显得支离破碎。AI 上还是这副样子：外界对 Google AI 产品的观感，从「叹为观止」一路跨到「这玩意儿当初是怎么想的就放出来了」。各块拼图之间的深度打通，看起来也是个难题。

最后，**Microsoft** 的企业级终局或许最稳，尽管不总是一眼能看出来。它的优势不在模型层——这一层它把自家几个专用模型和 OpenAI、近来还有 Anthropic 的大模型混着用。真正的底气在于：企业里工作真正发生的那些地方，Microsoft 早就攥在手里——Microsoft Office/365、Teams、GitHub 等等。Copilot 在市场上算不上最强的聊天产品，却照样无处不在。

Microsoft 也在用 Foundry、Copilot Studio 和各类 Azure 产品猛攻运行时和构建层。只是要在它那套盘根错节的产品里找到路，本身就够费劲。

## 它们到底在抢什么

表面看，这些厂商都想从 B2B 客户那儿多分一杯羹。没错，但这只是半幅画面。它们真正在抢的，是成为 AI 工作默认发生的那个环境。

成为工作的默认平台，回报可以非常惊人。问问 Bill Gates 就知道了。

回看以往的平台战，赢家通常握着控制力和价值创造最高的那个点。PC 时代是 CPU 加 OS 的标准栈，这又让 Microsoft 顺势统治了生产力软件；移动时代是操作系统和应用商店；云时代是控制面和服务广度。

AI 里这个「咽喉点」是什么，现在还没人说得准。一度看起来模型能力会决定胜负，但如今几家大玩家的水平已经咬得很近，这条恐怕不成立了。在 B2B 场景，答案可能是运行时、上下文、治理三者的组合，再与最好的模型紧密咬合。

OpenAI 和 Anthropic 现在训练最新模型时，都跟自家的 agent harness 紧密绑定。这是垂直整合的明确信号——驱动力不只是商业算计，还有技术性能本身。

最终胜出的平台，会是那个让「可信、互联、嵌进真实工作流」的 agent 部署起来最省心的平台。这也是为什么我认为治理值得被当成头等的平台层来对待。四家都在谈治理，其中 Microsoft 经验大概最足。久而久之，治理可能成为这场平台战里最不起眼、却最关键的一块。

## 其他玩家呢

想当「跑你 agent 的平台」之主的，当然不止上面这四家。还有不少玩家也在场，但相比之下明显弱一截——它们攥住的栈不够多。

AWS 把基础设施做得很扎实，却在模型和 UI 上没什么存在感。Salesforce 在把自家 agent 基础设施作为核心产品线的延伸来推，但这对非 Salesforce 客户没多大吸引力。SAP、Oracle 和其他老牌厂商，处境也差不多。

再往下，还有一批专攻某一层、做出同类最佳产品的小玩家。

这是个难受的位置。看看 Cursor 就明白了——几个月之间，它从人人追捧的编码工具，沦落到几乎没人再提。和平台所有者顺手提供的、还挺能打的工具竞争，难。

## 接口会开放还是封闭

和早年的平台战相比，这一次有个有意思的不同：哪怕平台在 UX、治理、模型层面贴身竞争，AI agent 栈底层的管道和集成点，也许会走向标准化和开放。

历来，平台都想牢牢掌控自己的内部运作，并精确划定第三方能在哪儿插足。App Store 上架你的 app 得苹果点头；Windows 应用能用哪些 API，Microsoft 说了算；云厂商想方设法把你引向它们打包好的服务，而不是让你拿开源组件自己搭。归根结底全是控制和客户锁定。

AI 这边，早早就冒出一批开放标准。MCP 让外部工具和数据得以接入，几家主流玩家都在支持；Google 把自家的 Agent2Agent 协议作为开放标准推出；skill 在平台之间迁移眼下也相当容易——我常在 Codex（编码能力最强）里开发 skill，然后丢到 Claude 或 OpenClaw 里跑，顺得出奇。

这种局面会一直持续吗？我看悬。以往平台战里的大赢家，都是一边用开放标准抢市场份额，一边靠自家专有方案攥住价值。偶尔也有彻底开放的栈笑到最后（比如做网站的 LAMP），但很罕见。

## 这对创业公司意味着什么

对创业公司来说，「在前沿模型上套个漂亮壳」或者「做个在幕后撑起酷功能的基础设施工具」就能轻松起家的好日子，明摆着要结束了。大平台的野心铺得极广——而且考虑到桌上押注的钱，它们不得不这么野。

眼下很多创业公司把自己定位成所谓的「平台补全者」：去填大平台产品线里的窟窿，专挑巨头还没拿得出手的地方下手——工作流引擎、agent 编排器、LLM 安全、记忆平台等等。这些都是好点子，市场需求也实打实。正因如此，其中一些公司能跑得飞快。

但从历史看，这是个危险的位置。如果年纪在 35 岁上下或更小，大概没听过 Borland、PowerBuilder、Lotus Development、Ashton Tate、Sybase 这些名字。它们都是 80 年代极其成功的软件公司，靠填补 Microsoft 和 IBM 产品线的窟窿起家——等平台所有者侵入它们的地盘，它们就大多消失了。

当然，就算身处大平台林立的世界，照样能做出成功的公司。比如 Snowflake 和 Databricks，在大云厂商都有竞品的情况下，仍成了数据仓库领域可以说最重要的玩家。只要真握有一款同类最佳、宽到不会沦为别人路线图上一个功能的产品，空间就不小。

另一条成功路线，是在某个相关层上做跨平台供应商。最有名的例子是 Oracle：正因为能跑在所有大型计算平台上，它才成了最重要的数据库厂商。

AI 里大概率会上演相似的剧情。能长久立住的机会，要么在平台之上，要么在平台之间：垂直 agent、专有工作流 UX、领域专属上下文、治理与合规、专门化的编排、多平台抽象层。最危险的位置在中间——那些只依附单一平台、自己又不掌握更广阔上下文的薄应用或基础设施功能。

这正是平台战的作用：把无差别层的价值压扁，把控制点的价值抬高。

创业公司应该好好研究平台的历史。在市场这个新阶段里怎么给自己定位，将决定成败。
